Exercise 15 — CSS Debug Challenge

Find and fix 6 intentional CSS layout bugs. Each bug is documented below with the broken code and the correct fix.

✅ Bug 1 Fixed
Missing box-sizing Reset
Without box-sizing: border-box, padding adds extra width outside the defined width, breaking layout calculations and causing overflow.
❌ Broken:
* { margin: 0; padding: 0; }
/* No box-sizing rule! */
✅ Fixed:
*, *::before, *::after {
box-sizing: border-box;
}
✅ Bug 2 Fixed
Header Not Sticky
The header used position: relative which causes it to scroll away with the page content instead of remaining fixed at the top of the viewport.
❌ Broken:
.header {
position: relative;
}
✅ Fixed:
.header {
position: sticky;
top: 0;
z-index: 100;
}
✅ Bug 3 Fixed
Fixed Grid Columns Overflow
Using fixed 300px columns prevents the grid from adapting to different screen sizes, causing horizontal scrollbars on mobile and tablet viewports.
❌ Broken:
grid-template-columns:
repeat(3, 300px);
✅ Fixed:
grid-template-columns:
repeat(3, 1fr);
✅ Bug 4 Fixed
Flexbox Vertical Misalignment
Using align-items: flex-start in a flex row causes items of different heights to snap to the top edge instead of centering vertically.
❌ Broken:
.flex-demo {
align-items: flex-start;
}
✅ Fixed:
.flex-demo {
align-items: center;
}
✅ Bug 5 Fixed
z-index: -1 Hides Element
Using z-index: -1 on an absolutely-positioned element causes it to render behind the container background, making it completely invisible to the user.
❌ Broken:
.zindex-back {
z-index: -1;
}
✅ Fixed:
.zindex-back {
z-index: 1;
}
✅ Bug 6 Fixed
Overflow: visible Breaks Layout
Using overflow: visible allows text content to bleed outside its container and creates unwanted horizontal page scroll on narrow screens.
❌ Broken:
.text-box {
overflow: visible;
}
✅ Fixed:
.text-box {
overflow: hidden;
text-overflow: ellipsis;
white-space: nowrap;
}
