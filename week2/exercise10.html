Bugs 1 & 2

Constant Reassignment & Missing Async Await

Bug 1 (Const Reassignment): Modifying variables declared with const throws a TypeError. We must use let if the reference will change.
Bug 2 (Async Promise Resolution): Async functions return a Promise. Trying to read results directly yields the Promise object instead of the resolved data. We must place the await keyword before the function call (and mark the wrapper function as async).

Broken Code
function renderCampaignHeader(id) {
  const defaultGoal = 100000;
  const totalGoal = defaultGoal;
  const dbGoal = getCampaignGoal(id);
  if (dbGoal > 0) {
    totalGoal = dbGoal; // Crashes
  }
  return "Goal: $" + totalGoal;
}
Fixed Code
async function renderCampaignHeader(id) {
  const defaultGoal = 100000;
  let totalGoal = defaultGoal;
  const dbGoal = await getCampaignGoal(id);
  if (dbGoal > 0) {
    totalGoal = dbGoal; // Works!
  }
  return "Goal: $" + totalGoal;
}
Bug 3

Implicit Global Variable Leakage

Omitting variable declarations (var, let, const) inside loops implicitly attaches the iterator variable i to the global scope (window or global). This causes namespace pollution and leads to unexpected side-effects or infinite loops if nested functions run.

Broken Code
function displayInvestorList() {
  let output = "";
  for (i = 0; i < MOCK.length; i++) {
    output += (i + 1) + ". " + MOCK[i];
  }
  return output;
}
Fixed Code
function displayInvestorList() {
  let output = "";
  for (let i = 0; i < MOCK.length; i++) {
    output += (i + 1) + ". " + MOCK[i];
  }
  return output;
}
Bug 4

Array Reference Comparison

In JavaScript, objects and arrays are compared by **reference**, not by value. Declaring tags === [] compares the array pointer against a brand new empty array pointer, which is **always false**. To verify if an array is empty, we must check its length: tags.length === 0.

Broken Code
function checkCampaignTags(id) {
  const tags = getCampaignTags(id);
  if (tags === []) {
    return "No tags assigned";
  }
  return "Tags: " + tags.join(", ");
}
Fixed Code
function checkCampaignTags(id) {
  const tags = getCampaignTags(id);
  if (tags.length === 0) {
    return "No tags assigned";
  }
  return "Tags: " + tags.join(", ");
}
Bug 5

Unhandled Async/Promise Rejections

Awaiting an async operation that rejects throws a runtime error. If not wrapped in a try/catch block, the exception bubbles up, resulting in an "Unhandled Promise Rejection" warning, which can crash Node environments or freeze UI execution.

Broken Code
async function executePurchase(userId, count) {
  console.log("Initiating...");
  await saveSharesToDatabase(userId, count);
  return "Purchase successful!";
}
Fixed Code
async function executePurchase(userId, count) {
  console.log("Initiating...");
  try {
    await saveSharesToDatabase(userId, count);
    return "Purchase successful!";
  } catch (error) {
    return "Error: " + error.message;
  }
}
